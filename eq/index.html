You can move by calling player.move(direction). For example: 

class Game {
    constructor(player) {
        player.move("south");
        player.move("east");
    }
}
The only parameters move will accept is "north", "east", "south", and "west".
You can perform recon by using player.look(direction). If you're unsure what a specific tile is, it will tell you. It's also a great way of making sure the enemy you're about to attack is challenging so you can run instead!
Like move, the look method also only supports "north", "east", "south", and "west".
To fight an enemy, simply make sure you're on a tile around the enemy, and use player.attack("enemyname"). As long as it's one tile away, the player will find it and start attacking immediately.
Be careful where you're walking, as landing on the same tile as an enemy will end you immediately.

Once the attack method is called, the player will begin attacking automatically. The chance of success is based on your attack power and enemies defence power. If the enemy rolls higher than you, it will attempt to attack you, in which case it takes your defence power into consideration.
Be sure you look for objects that increase your attack/defence power to make fights quicker and easier.

Example Code 

class Game {
    constructor(player) {
        player.move("east");
        player.attack("crab");
    }
}
After some specific actions events are triggered. You can catch them by simply creating a method in your Game class to catch it.
As an example, you can issue a block of code after defeating an enemy in battle with onattackend as shown below.


class Game {
    constructor(player) {
        // store a copy of the player instance
        this.player = player;
        player.move("south");
        player.attack("crab");
    }

    onattackend(mob) {
        this.player.move("east");
        this.player.look("west");
        console.log("I hath slayed the mighty %s",
            mob);
    }
}
